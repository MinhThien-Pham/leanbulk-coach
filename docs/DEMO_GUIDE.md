# Demo Guide - LeanBulk Coach

This guide walks you through evaluating the LeanBulk Coach MVP in under 5 minutes.

## 2-Minute Quick Start (Docker Compose)

The easiest way to run the full application is using Docker Compose.

### Step 1: Start the services
From the repository root, run:
```bash
docker compose up --build
```
This builds and starts the backend API (port 8000) and the React frontend (port 5173).

### Step 2: Access the Frontend
Open your browser and navigate to:
[http://localhost:5173](http://localhost:5173)

### Step 3: Seed the Demo Profile
1. On the default **Onboarding** tab, locate the **Demo Profile** panel at the top.
2. Click the **Create Demo Profile** button.
3. This triggers a local seed workflow that creates a user profile, inserts historical bodyweight/waist logs, seeds workout sets and meal entries, and logs an active safety flag.
4. You will see a success summary displaying the seeded counts and the next actions. The app will automatically select this user ID and redirect you to the **Dashboard** tab.

### Step 4: Explore the Dashboard
1. Review the **Goal & Profile**, **Safety Status** (should show `attention_needed`), and **Current Targets** panels.
2. Look at the **Next Actions** banner. It will display recommendations generated by the deterministic coaching rules.
3. View the custom, CSS-rendered **Weight Trend** and **Waist Circumference Trend** charts showing 7 days of seeded logs.
4. Locate the **Open Safety Alert Flags** section. You will see a warning: `Demo flag: user reported knee discomfort during squats.`
5. Click **Resolve** next to the flag. The dashboard will automatically refresh, the warning will disappear, and the **Safety Status** will update to `clear`.

### Step 5: Try the Meal Helper
1. Switch to the **Meal Helper** tab.
2. The page automatically loads the targets and remaining daily macros based on today's logged meals.
3. Click the **Get Meal Suggestions** button.
4. The panel displays 2-3 deterministic recipe ideas fitting the remaining calorie and protein gaps, respecting your kitchen equipment and dietary preferences.

### Step 6: Verify the Evaluation Suite
1. Switch to the **Dev Panel** tab.
2. Scroll to the **Evaluation Suite** section.
3. Click **List Eval Cases** to view all configured test cases.
4. Click **Run Eval Suite** to run all safety and regression evals. You should see a score of `100.0%` with `12/12 cases passed`.
5. Click **Show Eval Report** to read the formatted regression report.

## Alternative: API-Only Demo (cURL)

If you prefer to verify the backend using command-line tools, you can use these commands while the backend is running at `http://localhost:8000`.

### 1. Check health status
```bash
curl http://localhost:8000/health
```
*Expected Outcome:* Returns API version, service status, and confirmation that `live_llm_calls` are disabled (`false`).

### 2. Seed a demo profile
```bash
curl -X POST http://localhost:8000/seed/demo-profile
```
*Expected Outcome:* Creates user profile ID `1` and returns the JSON payload summarizing the seeded entries.

### 3. Fetch user contexts
```bash
curl http://localhost:8000/context/1
```
*Expected Outcome:* Fetches aggregated profile, body history, target nutrition, and safety flag contexts.

### 4. Build coaching summary
Send the contexts retrieved from the step above to get the coach summary:
```bash
curl -X POST -H "Content-Type: application/json" -d '{
  "profile_context": {"id": 1, "display_name": "Demo User", "sex": "male", "age": 25, "height_cm": 180, "goal": "lean_bulk"},
  "latest_body_context": {"weight_kg": 75.5, "waist_cm": 80.1},
  "body_history_context": [],
  "latest_nutrition_context": {"target_kcal": 2700, "protein_g": 150},
  "recent_workout_context": [],
  "recent_meal_context": [],
  "open_safety_context": [],
  "progress_summary_context": {"weight_trend": "insufficient_data"}
}' http://localhost:8000/summary/build
```
*Expected Outcome:* Returns the deterministic coach summary containing targets, statuses, and next actions.
