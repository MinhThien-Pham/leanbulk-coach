# Demo Script — LeanBulk Coach

**Version**: 2.0  
**Purpose**: Step-by-step walkthrough for reviewer/judge evaluation of the local deterministic LeanBulk Coach demo.

---

## Overview

The LeanBulk Coach MVP runs **fully offline**. All calculations, trend analysis, and safety checks are performed deterministically by local Python tools.

---

## Prerequisites

- Ensure Docker is running.
- Start the application using:
  ```bash
  docker compose up --build
  ```
- Backend will be available at `http://localhost:8000/health`.
- Frontend will be available at `http://localhost:5173`.

---

## 2-Minute Demo Flow

### Step 1 — Seed the Demo Profile (30 seconds)
1. Navigate to the Frontend UI at `http://localhost:5173`.
2. On the default **Onboarding** tab, locate the **Demo Profile** panel.
3. Click the **Create Demo Profile** button.
4. This seeds the local SQLite database with a mock user history (profile `Demo LeanBulk User`, 7 chronological weight/waist logs, 4 workout logs, 3 meals, and 1 safety flag).
5. The frontend will automatically load the seeded user ID and swap the active tab to the **Dashboard**.

### Step 2 — Explore the Dashboard (30 seconds)
1. Review the **Goal & Profile** and **Current Targets** panels.
2. Observe the custom CSS-rendered **Weight Trend** and **Waist Circumference Trend** charts displaying the 7 days of seeded weight logs.
3. View the **Next Actions** banner generated by the coaching rules.

### Step 3 — Resolve Safety Alerts (30 seconds)
1. Locate the **Open Safety Alert Flags** section displaying: `Demo flag: user reported knee discomfort during squats.`
2. Notice that the **Safety Status** badge indicates `attention_needed`.
3. Click the **Resolve** button next to the safety flag.
4. The dashboard automatically refreshes: the alert disappears, and the **Safety Status** badge updates to `clear`.

### Step 4 — Try the Meal Helper (30 seconds)
1. Switch to the **Meal Helper** tab.
2. The page automatically fetches the daily target and loads the remaining calorie/protein targets.
3. Click **Get Meal Suggestions**.
4. Explore the recipe ideas generated based on your kitchen equipment and dietary preferences.

### Step 5 — Run the Evaluation Suite (30 seconds)
1. Switch to the **Dev Panel** tab.
2. Scroll to the **Evaluation Suite** section.
3. Click **Run Eval Suite** to run all safety and regression tests.
4. Observe that the evaluation completes with a score of `100.0%` and `12/12 cases passed`.
5. Click **Show Eval Report** to read the formatted regression report.

## Optional: API-Only Tour (cURL)

Graders can also verify the endpoints directly:

1. **Check health status:**
   ```bash
   curl http://localhost:8000/health
   ```
2. **Seed a demo profile:**
   ```bash
   curl -X POST http://localhost:8000/seed/demo-profile
   ```
3. **Fetch aggregated context:**
   ```bash
   curl http://localhost:8000/context/1
   ```
4. **Calculate calorie target:**
   ```bash
   curl -X POST -H "Content-Type: application/json" -d '{
     "weight_kg": 75.0,
     "height_cm": 180.0,
     "age": 25,
     "sex": "male",
     "activity_level": "moderately_active",
     "goal": "lean_bulk"
   }' http://localhost:8000/tools/calorie-target
   ```

## Key Concepts to Call Out
- **Fully Offline & Safe:** No external calls are made, avoiding rate-limiting or network issues during grading.
- **Rule-Based Safety:** Calorie thresholds and safety alerts are calculated deterministically to prevent extreme diet advice.
- **Deterministic Evals:** Graders can run `pytest --tb=short -q` or `python -m backend.evals.runner` locally to verify quality gates.
